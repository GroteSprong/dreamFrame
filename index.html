Content layout

The layout for the content of a page is structured in so called blocks: a SECTION element with a centered ARTICLE element inside. Standard way of building blocks. The ARTICLE will take care of equal content width everywhere.

PUT ALL CONTENT WITHIN THE ARTICLE BLOCK
MARGINS AND PADDINGS ARE TAKING CARE OF

Use background colors or background images to layout the blocks...

CSS Custom Properties

Also known as CSS varibles. Set all the variables for fonts, colors, widths ect in the top of the CSS file.
You can set for colors and dimentions for all main elements of the page (navigation, header, main content area, sidebars, footer.)

These custom properties will effect the whole site, so, all pages. (For individual pages you can set the custom properties to an other value if you want a special page)

Create and apply your own color schemes, background images, block layout, font settings.

Attribuut selectors

we use atribute selectors, using the data- prefix. Why not using classes like all do?
Attribute selectors can be easy manupilated with Javascript. They can be more simple than using classes, while, no value is needed. Just one phrase will do. On the other hand, if we use values, it can be more intuitive versus using multiple classes inside a tag.

Simple "modules"

Any kind of stuff that makes life easier. From two-column gadget to video player.

Look at the block at the top of the page: Text on the left side, image on the right. It can be achieved bij adding an attribute selector to an article within a section: data-gs-twocol. On smaller screens, the columns will dissapear and the content will be stacked.
