Every page can have a navigation (nav), a main section with all the content (main), and a footer (footer). Normally this will be the case, but for landing pages and such you could leave navigation and footer out if you want.

The navigation and the footer will be loaded by a javascript file, you can find the script tag under the footer. For that, check out the navigation and Javascript docs.

So this means every page will automagically load the same navigation and footer. (If you would like to load some other navigation or footer on some of the pages, this is very simple to accomplish).
In this way, we have to make the navigation and footer only once, and it will appear on all pages, and will change on all pages if adjusted.

The actual content of the page will always be inside the main element. This way, we can always load other pages inside this main element, to dynamically load pages without touching the further parts (navigation, footer). We can also load external content in this main element.

Main content

If no colums are needed, pages can be build like blocks of content that will be placed after, in other words beneath each other. Much like Bootstrap works, only with simpler code.

The main section contains the content of the page. It is structured in blocks that are placed beneath each other.

A block consists of two containers: a section element that has a 100% width and serves as a wrapper, and an article element with the content of the block. It will have a standard width of 80.9% width of the screen, which is based on the golden ratio.

Because we can load pages with javascript dynamically, we could work with one master (template) file and load all the others in the MAIN section of that page.
We can also load external pages from other websites in the main area (if CORS rules are met). More on this in the Javascript doc.
