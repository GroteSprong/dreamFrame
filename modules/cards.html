Cards:

  • Small content boxes
  • Made up of a header, content area and footer
  • Placed next to each other.
  • Equal height regardless content.
  • Responsive: a card will go to a new row when not enough space.
  • Customise with attribute selectors

Code explain

Put in one SECTION element with an attribute: data-gs-cardgroup

Inside that, put a DIV with an attribute: data-gs-card

Inside that div, place HEADER, SECTION and FOOTER elements.
Add more DIVs with data-gs-card to add more cards to the group. Be sure they are inside the SECTION element with the data-gs-cardgroup attribute.

the Cardgroup CSS

All we need for the group is a display of "flex", so the cards will be next to each other by default. Flex-wrap is set to "wrap", so that, if there's not enough room, the flex element (the card) will be placed on the next line.

Card CSS

the card CSS

The card is also a flex element. It has the direction "column", so its items will be placed underneath each other, not next to each other. The children are the header, content and footer.

third line: justify-content: space-between;
This means the items will be put straight against top and bottom, with equal space between. This means the header will be forced to the top of the card, and the footer to the bottom.

flex: 1 0 var(--card-min-width);
This is saying three things: the card will grow if possible, but it will not shrink. And the var tells us what the minimum width of the card should be. You can set this var yourself in top of the stylesheet. Default setting is 15em. This means, a card cannot be smaller than 15em. If there's not enough room, the card will be placed on the next line.

Background-color, border and margin you can set also in the vars at the top of the CSS file.
